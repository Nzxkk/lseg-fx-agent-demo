# FX LLM Report Writer

Use this skill when the user asks for a Chinese business report generated by a large language model.

## Source Boundary

- The LLM may explain and summarize computed results.
- The LLM must not invent prices, news, signal direction, model performance, or missing data.
- The deterministic signal engine remains the source of truth for `side`, `composite_score`, and `confidence`.

## Tool Flow

1. Read latest signal CSV.
2. Read latest news CSV.
3. Read backtest summary.
4. Read Agent decisions and risk checks.
5. Send compact structured context to the configured LLM endpoint.
6. Save the generated report to `fx_macro_news_llm_report.md`.

## Configuration

- Default provider: Qwen / DashScope OpenAI-compatible endpoint.
- Default model: `qwen-plus`.
- API key should be entered in the page or provided through environment variables.
- Do not write API keys into source files or reports.

## Output Contract

The report should contain:

- Today conclusion
- Per-instrument interpretation
- News and macro impact
- Risk caveats
- Next observation points

The report must remain clearly research-only.
